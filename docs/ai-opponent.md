# AI 상대는 어떻게 게임을 진행하는가

이 문서는 컴퓨터 상대(`TAiPlayer`, `src/engine/Gostop.AI.pas`)가 매 턴 어떤 논리로 카드를 내고
고/스톱을 판단하는지 정리한 것이다. 엔진과 어긋나면 코드가 아니라 문서를 의심할 것.

---

## 1. 한 줄 요약

**능력치(Skill, 0~100) 값 하나**가 다음 4가지 축을 동시에 조절한다.

| 축 | 낮을 때 | 높을 때 |
|----|---------|---------|
| 실수·무작위성 | 최선수 대신 무작위 수를 자주 선택 | 거의 항상 최선수를 선택 |
| 수읽기 깊이 | 순수 1-플라이 휴리스틱만 사용 | 결정화 몬테카를로로 여러 수 앞까지 시뮬레이션 |
| 방어·견제 가중 | 내 득점만 고려 | 상대 위협도를 함께 고려해 견제 수를 섞음 |
| 고/스톱 판단 정밀도 | 단순 휴리스틱 + 확률적 흔들림 | "지금 스톱" vs "고" 기대 손익을 시뮬레이션으로 비교 |

이와 별개로 **배짱(GoBias)**과 **욕심(Greed)**은 능력치와 무관하게 캐릭터마다 고유값을 가진다
(4절 참고). 즉 "이 AI가 얼마나 잘 두는가"는 능력치가, "이 AI가 어떤 성격으로 두는가"는
배짱·욕심이 담당한다.

---

## 2. 난이도 선택과 실제 적용 범위

타이틀 → 새게임 설정에서 고르는 난이도는 4단계뿐이다(`Gostop.Board.pas`의 `AI_SKILL_LABELS`/`AI_SKILL_VALUES`):

| 표시 이름 | 능력치(Skill) |
|-----------|--------------:|
| 병아리 | 30 |
| 선수 | 50 |
| 타짜 | 70 |
| 신의손 | 100 |

- 여기서 고른 능력치는 **그 판의 모든 AI 좌석에 동일하게 적용**된다(`FConfig.AiSkill` → 모든 좌석의 `FSeatSkill`).
- 반면 **배짱·욕심은 항상 캐릭터 고유값**이며 난이도 선택과 무관하다(`TGostopCharacters.NerveBias`/`GreedBias`를
  좌석 배정 시 개별 적용).
- `Gostop.Characters.pas`에는 캐릭터별 "유도 스킬"(`DerivedSkill` = (수읽기+침착)×1.25)을 계산하는 함수가
  있고, `docs/characters.md`는 이를 난이도 "[고유]" 옵션으로 쓴다고 설명하지만 — **현재 코드에는 이 값을
  실제 난이도에 적용하는 지점이 없다(미배선 상태)**. 실제로는 위 4단계 중 하나만 전 좌석에 균일 적용된다.
- 캐릭터의 "운(Luck)" 스탯도 `characters.json`에 저장은 되지만, AI 판단 로직 어디에서도 읽지 않는다(미구현).

## 3. 매 턴 흐름 — `Act` → `DoPlay` / `DoGoStop`

```
TTurnEngine이 AI 차례가 되면 TAiPlayer.Act 호출
├─ Phase = gpPlaying        → DoPlay   (카드 내기/폭탄/뒤집기)
└─ Phase = gpAwaitingGoStop → DoGoStop (고/스톱 선언)
```

### 3-1. `DoPlay` — 카드 내기

1. 손패가 0장(카드빚)이면 무조건 `FlipOnly`(강제, 판단 없음).
2. `GenerateMoves`로 후보 수를 만든다:
   - 손패 각 장을 냈을 때의 평가값(`EvaluateHandMove`) — 손패 수만큼.
   - 지금 낼 수 있는 폭탄(같은 월 3장 손패 + 바닥에도 같은 월)이 있으면 그 수도 후보에 추가.
3. **능력치로 정해지는 시뮬레이션 횟수(`SimCount` = round(능력치/100 × 12), 최대 12)가 0이면**
   → 1-플라이 휴리스틱: `GreedyBest`(평가값 최댓값)로 최선수를 고른 뒤 `PickWithMistakes`로 확률적으로
   무작위 수와 섞는다.
4. **`SimCount` > 0이면(능력치가 있으면)** → 상위 `TOP_K`(3)개 후보만 추려 결정화 몬테카를로로
   정밀 평가(5절)한 뒤, 그 결과에도 다시 `PickWithMistakes`를 적용해 최종 실행.

### 3-2. `DoGoStop` — 고/스톱 판단

- **능력치 0(`SimCount`=0)**: 손패 장수(회복 여력)·상대 위협도·배짱 보정을 섞은 휴리스틱으로
  "고를 원하는지" 판정한 뒤, 능력치에 비례한 확률로 그 판단을 따르거나 그냥 동전 던지듯 무작위로 정한다.
  단, 이미 7점 이상이면 고를 원하지 않는 쪽으로 기운다.
- **능력치 > 0**: "지금 스톱"했을 때의 확정 순손익과, "고"를 선택하고 `SimCount`회 몬테카를로
  롤아웃한 평균 순손익(배짱 보정 포함)을 비교해 더 큰 쪽을 선택한다.

## 4. 카드 가치 평가 (`CardValue`)

1-플라이 휴리스틱과 방어 가중치 계산의 기반이 되는 카드별 점수:

| 종류 | 값 |
|------|---:|
| 광 | 20 |
| 열끗 · 고도리(2·4·8월) | 13 |
| 열끗 · 국진(9월) | 11 |
| 열끗 · 그 외 | 8 |
| 띠 · 무늬 있음(홍단/청단/초단) | 8 |
| 띠 · 무늬 없음 | 5 |
| 피 · 쌍피 이상 | 4 |
| 피 · 단피 | 2 |
| 보너스패 | 2 + 피값 |
| 그 외 | 1 |

`EvaluateHandMove`는 이 값에 "바닥에 몇 장이 매칭되는지", "욕심(Greed)이 낮을수록 커지는 방어 가중치
(`LDefWeight` = 능력치 × 상대위협/10 × (100−욕심)/50)"를 더해 손패 한 장을 냈을 때의 점수를 낸다.
바닥에 매칭이 없으면(그냥 내려놓기) 카드 값어치만큼 마이너스로 평가해 "아까운 패는 덜 내는" 경향을 만든다.

`OpponentThreat`은 다른 플레이어들이 이미 먹은 패의 족보 점수(광 2·4장 보유 시 가산)를 계산해 그중 최댓값을
"지금 얼마나 위험한 상대가 있는가"로 쓴다.

## 5. 결정화 몬테카를로 수읽기 (능력치가 있을 때만)

실제 고스톱은 상대 손패와 남은 뒷패 순서를 모르는 불완전정보 게임이라, AI는 능력치가 있을 때
매번 **"안 보이는 카드를 그럴듯하게 무작위 배치한 가상 세계"**를 만들어 그 안에서 게임 끝까지
시뮬레이션해보고, 그 결과(실제 정산 손익)를 후보 수의 점수로 쓴다. 능력치가 0이면 이 과정 자체를
건너뛰고 1-플라이 휴리스틱만 쓴다.

알고리즘 단계, 파라미터, 한계(상대 모델링 부재 등)에 대한 자세한 내용은
[`docs/ai-monte-carlo.md`](ai-monte-carlo.md) 참고.

---

## 6. 결정론적 재현성

AI는 시드 기반 LCG(선형합동생성기)로 난수를 만든다(`NextRandom`/`NextFloat`). 좌석마다 서로 다른 시드로
생성되므로, 같은 시드로 다시 실행하면 항상 같은 수를 두는 결정론적 동작이 된다 — 디버깅과 밸런스
튜닝에 유리하다.

---

## 7. 능력치·성향이 적용되지 않는 결정들

아래는 겉보기엔 "AI의 판단"이지만, 실제로는 이 문서의 능력치/몬테카를로 로직을 거치지 않고
**단순 고정 규칙 또는 확률**로 처리된다:

- **4인 광팔기 협상**: 참가(P2·P3 역할 AI)는 항상 참가하도록 고정, 판매 여부(P4 역할 AI)는
  "광값이 0보다 크면 판매"라는 단순 조건 하나뿐 — 몬테카를로 평가 없음.
- **쇼당(3인) 걸기**: 조건이 성립하면(`TShodang.Detect(...).Callable`) 능력치와 무관하게 무조건 건다.
- **쇼당 수락/거절**: 관전 모드에서 AI끼리 자동 종합할 때는 `Random(100) < 60`, 즉 그냥 60% 확률로
  수락 — 능력치·배짱·욕심 전부 관여하지 않는다.
